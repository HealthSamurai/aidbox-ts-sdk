### Правила мэппинга Figma → дизайн‑система (react-components)

Эти правила обязательны при любой работе по синхронизации макетов Figma с кодом пакета `react-components`.

## 1) Источник истины: токены только из @theme в `src/index.css`

- **Всегда** используем только токены из секции `@theme { ... }` файла `packages/react-components/src/index.css`.
- **1:1 соответствие имен**: переменные Figma маппим на такие же по смыслу токены из `@theme` (не используем произвольные/альтернативные токены).
- **Никаких raw‑значений**: не вставляем прямые HEX/RGB/px и т.п., только классы/переменные, завязанные на `@theme`.
- **Примеры корректных ссылок**:
  - Текст: `text-text-primary`, `text-text-tertiary`, `text-text-link`.
  - Фон: `bg-bg-primary`, `bg-bg-tertiary`.
  - Элеенты на фоне (кроме текстов и бордеров): `fg-fg-primary`, `fg-fg-tertiary`.
  - Бордер: `border-border-primary`.
  - Фокус/кольцо: `focus-visible:ring-ring` (если такой токен есть и используется в проекте).

## 2) Типографика: только пресеты из `src/typography.css`

- Любые текстовые стили берём **из готовых пресетов** в `packages/react-components/src/typography.css`.
- Не собираем типографику из utility‑классов (`text-xl`, `leading-7`, `font-medium`, ...).
- Если пресета нет — добавляем новый, используя **только** токены из `@theme`/`tokens.css`.
- Примеры существующих пресетов:
  - `.typo-body` — Inter 14/20 Regular.
  - `.typo-label` — Inter 14/20 Medium.
  - `.typo-page-header` — Inter 20/28 Medium + `letter-spacing: var(--font-tracking-tight)`.

## 3) Letter‑spacing из Figma → `--font-tracking-*` в `src/tokens.css`

- Всегда маппим `letter-spacing` (tracking) на переменные из `packages/react-components/src/tokens.css`:
  - `--font-tracking-tighter`, `--font-tracking-tight`, `--font-tracking-normal`, `--font-tracking-wide`, ...
- Применяем letter‑spacing исключительно через пресет в `typography.css`.

## 4) Процесс работы с макетом Figma

1. Получили ссылку/узел → вытягиваем переменные/типографику.
2. Сопоставляем цвета/фоны/бордеры строго с токенами из `@theme` `src/index.css` и используем соответствующие классы (`text-…`, `bg-…`, `border-…`).
3. Типографику сопоставляем с пресетами из `typography.css` с префиксом `typo-`. Если нет — добавляем новый пресет с токенами из `@theme`/`tokens.css`.
4. Вносим правки **только в стили**, не меняя логику/структуру компонентов.
5. Если нет 1:1 соответствия или есть сомнения — останавливаемся и запрашиваем уточнение.

## 5) Что нельзя

- Нельзя использовать значения не из `@theme` (включая переменные вне секции `@theme`, произвольные CSS‑переменные, HEX/RGB и т.д.).
- Нельзя собирать типографику из utility‑классов вместо пресетов.
- Нельзя менять поведение/логику компонента при задачах на стили.

## 6) Пример (Breadcrumb)

- Цвета:
  - Заголовок (активная страница): `text-text-primary` + пресет `.page-header` (Inter 20/28 Medium + tracking tight).
  - Список/чипы (рядовых крошек): `text-text-tertiary` (серый из Figma) и фон `bg-bg-tertiary`.
  - Разделитель: `text-text-quternary`, размер `text-xs`.
- Типографика:
  - Список/линки: используем `.typo-body`.
  - Активная страница: `.typo-page-header`.
- Вёрстка:
  - Только классы (layout/spacing), логика/DOM‑структура без изменений.
  - Разделитель по умолчанию — символ `/`.

## 7) Нейминг классов, ссылающихся на токены `@theme`

- `text-…` → `var(--color-text-…)`
- `bg-…` → `var(--color-bg-…)`
- `border-…` → `var(--color-border-…)`
- Ховеры/состояния используют соответствующие токены `…_hover`, `…_on-brand` и т.д., если они есть в `@theme`.

## 8) Проверка качества

- Линтер без ошибок.
- Визуальное соответствие Figma, в том числе размеры, отступы, межбуквенные интервалы, состояния.
- Токены и пресеты используются корректно; отсутствие raw‑значений.
